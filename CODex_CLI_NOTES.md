# Codex CLI Notes (для моей работы после запуска CLI)

Если вы запускаете Codex CLI и хотите передать контроль мне, оставьте следующие данные/шаги прямо в чате CLI сразу после старта:

1. **Путь к проекту** — подтвердите, что CLI открыт в `telegram-ai-bot`. Если путь другой, сообщите точный каталог.
2. **Состояние `.env`** — скажите, какие файлы доступны: `.env`, `.env.prod`, `.env.local`. Если есть изменения, укажите ключевые значения (без раскрытия секретов: можно сообщить, что `YC_SEARCH_INDEX_ID` обновлён и т.п.).
3. **Текущие задачи** — сошлитесь на конкретный backlog ID (например, BL-02, BL-03) или скажите, какую задачу надо продолжить.
4. **Доступные ресурсы** — если есть новые ID контейнера, job, Lockbox, передайте их (пример: `CONTAINER_ID=e9q...`). Без этого я могу только подготовить команды.
5. **Команды, которые можно/нельзя запускать** — уточните, разрешён ли `docker`, есть ли доступ к `yc`, нужно ли использовать Cloud Shell или локальную машину.
6. **Последние лог-файлы/отчёты** — если уже есть `diag_report.md`, `monitoring_alerts.yaml` с заполненными ID или вывод команд, приведите их в CLI, чтобы я мог анализировать.
7. **Цель сессии** — кратко сформулируйте, что требуется за эту итерацию (например, “запустить Cloud Deploy”, “использовать diag_report.py и отправить отчёт”, “починить кириллицу в README”).

После того как вы предоставите эту информацию в CLI, я смогу продолжить работу (писать патчи, скрипты, инструкции).
Если нужно, добавьте любой другой контекст (ошибки, скриншоты, ссылки) — это ускорит процесс.

Напоминание: перед стартом сеанса прочитайте README.md — в нём описаны роли, принципы Agile/TDD/XP, расписание ретроспектив (каждые 5 ответов) и refinement (каждые 3 ответа).
