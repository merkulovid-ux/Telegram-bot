# GEMINI_START.md

Этот файл предназначен для Gemini-агента, который будет работать с проектом ProcessOff.

## 1. Общая информация перед началом работы
1. Убедитесь, что находитесь в корневой директории проекта (`telegram-ai-bot`).
2. Прочитайте `README.md` и `Yandex_INFRA_SETUP.md`, чтобы понять контекст Release 2 (Deployment Automation).
3. Изучите `BACKLOG.md` и `ROADMAP.md`: Release 2 в работе, Release 3 запланирован, ревью каждые 5 ответов, refinement каждые 3.
4. Проверьте `.env` / `.env.prod` (или другие файлы конфигурации) и убедитесь, что они не содержат секретов, которые должны быть в Lockbox (`python export_lockbox_payload.py ...`).
5. Убедитесь, что все тесты (`pytest tests/test_smoke.py`) и проверки (`python predeploy_check.py --env .env --env .env.prod`) проходят успешно.

## 2. Рабочий процесс
- **DevOps задачи (Release 2)**: Фокус на `Yandex_INFRA_SETUP.md` и `scripts/generate_yc_bootstrap.py`. В процессе работы будут генерироваться команды `yc`/`docker push` для настройки инфраструктуры.
- **RAG/LLM задачи**: Работа с `handlers/llm_commands.py`, `assistant_client.py`, `responses_client.py` для реализации функционала RAG, суммаризации (по необходимости) и других задач из backlog.
- **Документация**: Обновление `README.md`, `monitoring_alerts.md`, `Yandex_*` файлов для поддержания актуальности.

## 3. Что делать, если что-то пошло не так
- Проверьте все необходимые ID (DevOps): Lockbox `secret_id`, ID сервисных аккаунтов, Registry, Container ID.
- Если не хватает каких-то ID, запросите их. Если есть ошибки, сообщите о них.
- Убедитесь, что у вас есть доступ к Yandex Cloud и необходимые права; не пытайтесь угадывать или использовать неверные данные.

## 4. Код и стиль
- Python: PEP8, type hints, docstrings (Google style), используйте автоматическое форматирование кода.
- Пишите чистый/читаемый код, следуйте принципам DRY/KISS.
- В случае сомнений: всегда спрашивайте, прежде чем вносить изменения, чтобы избежать регрессий.

## 5. Коммуникация
- Всегда начинайте работу с обсуждения и уточнения задачи (refinement backlog).
- Регулярно проводите ревью и ретроспективы (согласно плану Release 2/3).
- Координируйте действия с DevOps/QA/Scrum Master, чтобы обеспечить бесперебойную работу.
